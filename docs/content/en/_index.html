{{< /blocks/cover >}} {{% blocks/lead color="primary" %}} OSPREY is computational protein design software developed by [the Donald Lab](https://www.cs.duke.edu/donaldlab/) at Duke University. It has been used to prospectively predict resistance mutations, design antibodies targeting HIV, and redirect enzymes to target new substrates, among other design and analysis tasks. A full listing of peer-reviewed papers that OSPREY has played an important role in can be found [here](www.todo.com). {{% /blocks/lead %}} {{< blocks/section color="dark" >}} {{% blocks/feature icon="fa-subscript" title="Provable bounds on partition functions" %}} OSPREY uses state of the art algorithms such as [MARK*](http://www.todo.com) to calculate bounds on partition functions {{% /blocks/feature %}} {{% blocks/feature icon="fa-bacon" title="Approximate binding" %}} Use OSPREY's K* algorithms to provably approximate binding a protein to a ligand {{% /blocks/feature %}} {{% blocks/feature icon="fa-atom" title="Probe stability" %}} Algorithms such as PAsTe provide insight into how mutations affect stability {{% /blocks/feature %}} {{< /blocks/section >}} {{< blocks/section >}} {{% blocks/feature icon="fa-laptop" title="Use a GUI" %}} OSPREY provides a slick, easy-to-use GUI for preparing your protein for design {{% /blocks/feature %}} {{% blocks/feature icon="fab fa-python" title="Use Python" %}} OSPREY also provides a Python interface for programmatically creating designs {{% /blocks/feature %}} {{% blocks/feature icon="fa-microchip" title="Use GPUs" %}} GPU codes speed up OSPREY designs by orders of magnitude. See the [benchmarks](https://doi.org/10.1101/306324). {{% /blocks/feature %}} {{< /blocks/section >}}
